DOM-Based XSS Demo

VULN: DOM-Based XSS (CWE-79)
The page reads location.hash and writes it directly to innerHTML. The payload never reaches the server — it bypasses all server-side input validation and WAF rules.

The page will display the URL fragment as a "welcome message".

Try: /vulnerable/dom-xss#<img src=x onerror=alert(document.cookie)>

Source of the vulnerability:

// Vulnerable JavaScript — reads location.hash and writes raw HTML
var fragment = decodeURIComponent(location.hash.slice(1));
if (fragment) {
    var el = document.getElementById('welcome-msg');
    el.style.display = 'block';
    el.innerHTML = 'Welcome: ' + fragment;  // ← sink
}
